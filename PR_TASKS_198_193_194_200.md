## Technical Details

### Architecture Decisions

1. **Analytics Data Source**: Currently computed from `getAllPrompts()`. The component is designed to easily swap to a dedicated analytics API when indexing is implemented.

2. **Network Detection**: Centralized in `networkDetection.ts` utility for reuse across components. Banner component is presentational only.

3. **Error Mapping**: Technical Stellar errors are mapped to user-friendly messages in a single utility function for consistency.

4. **Test Mocking**: All external dependencies (wallet, contract, unlock service) are mocked to avoid flaky tests and enable fast CI runs.

### Performance Considerations

- Analytics query cached for 30 seconds to reduce RPC calls
- Metadata fetched only when modal is open
- Network detection runs synchronously (no async overhead)
- Test suite runs in <2 seconds

### Accessibility

- Analytics cards have proper semantic structure
- Network banner uses appropriate ARIA roles
- Modal maintains keyboard navigation and focus management
- All interactive elements have proper labels
